# Ubiquitous Language

## Purpose

Keep dot-agent planning, implementation, review, and generated docs aligned on the same harness vocabulary.

## Terms

| Term | Definition | Use When | Compress / Aliases | Avoid | Evidence |
|------|------------|----------|--------------------|-------|----------|
| Harness | The personal agent system rooted at `~/.dot-agent` that packages runtime config, skills, docs, scripts, and local state for Codex and Claude. | Referring to the whole dot-agent system. | dot-agent system, agent harness | project, app, framework | `README.md`, `docs/harness-runtime-reference.md` |
| Runtime Home | Installed target directory consumed by a runtime, such as `~/.codex` or `~/.claude`. | Discussing installed config or copied/symlinked skill payloads. | runtime target, install target | source repo | `README.md`, `setup.sh` |
| Skill | A reusable workflow module under `skills/<name>/` with `SKILL.md`, `skill.toml`, and optional helper dirs. | Naming task-specific agent capabilities. | command, workflow module | plugin when no plugin exists | `skills/README.md`, `skills/AGENTS.md` |
| Skill Source | The tracked dot-agent skill directory that setup installs into runtime homes. | Distinguishing edited source from installed runtime payload. | source skill, shared skill source | runtime copy | `docs/harness-runtime-reference.md`, `skills/references/skill-authoring-contract.md` |
| Composes With | The seven-row ownership contract in each retained skill: parent, children, format, state reads, writes, handoffs, and returns. | Explaining how skills interact without duplicating ownership. | composition contract | dependency graph | `skills/README.md`, `skills/AGENTS.md` |
| Write Through | A rule that state mutation must go through the owning skill or deterministic helper. | Mutating roadmap, feature artifacts, reviews, or repo docs. | writes through, write gate | direct hidden mutation | `skills/README.md`, `skills/focus/SKILL.md` |
| State | Gitignored machine-local operating memory under `~/.dot-agent/state/`. | Referring to roadmap, ideas, daily reviews, execution reviews, or tool caches. | local state, operating memory | tracked docs | `README.md`, `docs/harness-runtime-reference.md` |
| Roadmap | The human day board at `state/collab/roadmap.md`, owned by `focus`. | Tracking active, review, parked, or blocked work. | focus board, day board | project/session database | `README.md`, `skills/focus/SKILL.md` |
| Feature Artifact | Durable planning files under `docs/artifacts/<feature>/`, owned by `spec-new-feature`. | Referencing spec, questions, research, design, and task files for non-trivial feature work. | feature docs, planning artifacts | idea doc, roadmap row | `skills/spec-new-feature/SKILL.md` |
| Handoff Packet | A self-contained Markdown packet under `docs/handoffs/` for outside review or remote continuation. | Packaging context for Research Pro or another reviewer. | review brief, external handoff | feature artifact | `skills/handoff-research-pro/SKILL.md` |
| Execution Review | Forensic review of Codex/Claude sessions and agent behavior, owned by `execution-review`. | Diagnosing session quality, verification, response fit, or workflow failures. | forensic review, session review | daily review | `skills/execution-review/SKILL.md` |
| Daily Review | Human day-end closure from roadmap state, owned by `daily-review`. | Recapping completed work and draining completed roadmap rows. | day-end review, closure recap | execution review | `skills/daily-review/SKILL.md` |
| Language Artifact | Repo-level shared vocabulary file at `docs/UBIQUITOUS_LANGUAGE.md`, owned by `ubiquitous-language`. | Aligning domain terms before planning, implementation, or review. | ubiquitous language doc, shared language doc | root glossary | `skills/ubiquitous-language/SKILL.md` |
